‚ö° Processing Pipeline

1
Customer Input & Safety Check
Customer submits complaint via React frontend ‚Üí Llama-Guard safety screening ‚Üí Input validation and preprocessing
2
Service & Actor Detection
LangChain orchestrator identifies Grab service (Food/Mart/Cabs/Express) ‚Üí Maps to appropriate actor (Customer/Driver/Restaurant/Dark Store)
3
Issue Classification & Routing
AI classifies complaint category ‚Üí Routes to specialized handler ‚Üí Determines if image evidence required
4
Context Enrichment & Validation
Fetches user order history ‚Üí Calculates credibility score ‚Üí Validates complaint against order status ‚Üí Gathers external API data (Maps/Weather)
5
AI Processing & Decision Making
Multi-model AI analysis (Text: Llama-3.3, Vision: Llama-4-Maverick) ‚Üí Fraud detection ‚Üí Resolution generation with GPT-OSS-120B
6
Predictive Analysis & Prevention
RAG model predicts potential future issues ‚Üí ML forecasting for proactive measures ‚Üí Performance protection application
7
Response Generation & Delivery
Personalized resolution generated ‚Üí Safety check on output ‚Üí Real-time delivery to customer ‚Üí Database logging for future reference
